M24.6 - Theme Authoring Reliability + Asset Health Pass

Base: M24.5.3

Scope:
- Theme authoring / baker workflow only.
- Steam lobby, P2P, holding room, portal, runtime level flow, and player networking files are intentionally unchanged.

Added:
- Per-scene-field Rebuild Wrapper buttons in the Theme Authoring form.
- Validation Report Asset Health section for every authored scene slot.
- Asset Health shows stored path, file existence, wrapper/source model alignment, root type, node count, mesh marker, and model instance marker.
- Wrapper scenes generated by M24.6 now include stronger metadata:
  - generated_by
  - generated_version
  - source_model_path
  - original_external_source_path
  - imported_model_res_path
  - generated_wrapper_res_path
  - m24_6_* metadata keys
- Save / Export / Validate status output is clearer and includes validation summaries.
- Managed import folders include README files so they survive zip packaging.

Notes:
- External .gltf/.glb import behavior from M24.5 remains in place.
- Raw .gltf/.glb paths are still not valid saved runtime scene paths; the theme data should store generated wrapper .tscn paths.
- External .tscn files are still rejected because arbitrary scene dependency copying is unsafe.
